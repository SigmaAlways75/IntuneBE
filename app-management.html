Win32 Delivery
Win32 App Delivery via IME — Full Flow

The Intune Management Extension (IME) handles Win32 apps, PowerShell scripts, and custom scripts on Windows — capabilities beyond what the core MDM protocol supports.

🔄 Complete Win32 Delivery Flow
1
Package with IntuneWinAppUtil.exe
Admin wraps the installer (EXE/MSI + dependencies) into a .intunewin file using Microsoft's packaging tool. Tool AES-256 encrypts the content and generates a metadata manifest. The key is stored in Intune, not in the package.
2
Upload to Intune (Azure Blob)
Admin uploads the .intunewin to Intune portal. Intune stores it in Azure Blob Storage and generates time-limited SAS URLs for content delivery. Admin configures detection rules, requirement rules, return codes, and dependencies.
3
IME Polls for Assignments
The Intune Management Extension (imewp.exe) runs as a Windows service. Polls Intune Graph API every hour (or on-demand) for new app assignments. Maintains a local queue of pending installs.
4
Requirement Rules Evaluated
Before downloading, IME evaluates requirement rules: minimum OS version, disk space, registry key exists, PowerShell script returns expected output. If requirements not met, installation deferred as "Not applicable."
5
Content Downloaded via SAS URL
IME downloads the encrypted .intunewin from Azure CDN using the time-limited SAS URL to C:\Program Files (x86)\Microsoft Intune Management Extension\Content\Incoming. Hash validated after download.
6
Decrypted and Extracted
IME fetches the AES-256 decryption key from Intune (using the device's MDM certificate for auth). Decrypts locally. Extracts installer and all dependency files to a temp working directory.
7
Install Command Executed
IME runs the install command (e.g., setup.exe /silent /norestart) as SYSTEM. Captures stdout/stderr and exit code. Custom return codes map to: Success, Soft Reboot, Hard Reboot, Retry, Failed.
8
Detection Rule Evaluated
After install, IME runs the detection rule: registry key, file version, MSI product code, or PowerShell script. If detection succeeds → "Installed" reported to Intune. If not → "Failed" with retry scheduled.
Query install status via Graph API
🔗
App Dependencies
Win32 apps can declare dependencies (e.g., VC++ Redistributable before main app). IME installs dependencies in order automatically. Dependency chains supported. Circular dependencies flagged as errors.
⬆️
App Supersedence
New version supersedes the old — Intune auto-uninstalls old and installs new on targeted devices. Tracks which devices have old vs new version. Gradual rollout via group rings.
Assignment rings
🔒
SYSTEM vs User Context
SYSTEM context: machine-wide install, no user login required, completely silent. User context: per-user install, requires user to be logged in. SYSTEM is most common and recommended for enterprise deployments.
Mobile App Stores
Apple VPP & Android Managed Google Play
🍎 Apple VPP — Volume Purchase Program
1. IT purchases app licences in Apple Business Manager (ABM)
2. Intune syncs with ABM via Apple VPP token — pulls licence inventory
3. Admin assigns app to a device or user group in Intune
4. Intune redeems a VPP licence token for each targeted device/user
5. Intune sends InstallApplication MDM command to device via APNs
6. App installs silently — no App Store sign-in, no prompts (supervised devices)
7. When user leaves, licence is automatically reclaimed and reassigned
iOS enrollment models
🤖 Android — Managed Google Play
1. Admin links Intune tenant to Google Enterprise via Managed Google Play iframe in Intune
2. Admin browses and approves apps in Managed Google Play — public or private apps
3. Intune syncs the approved app catalogue via Google EMM API
4. Admin assigns app to Android Enterprise device groups in Intune
5. Intune calls Google EMM API to push install → Google delivers via FCM to device
6. Android Device Policy app installs silently in the Work Profile
7. Private/LOB apps: uploaded directly to Managed Google Play as private apps

MAM SDK
App Protection Policies — Technical Detail

App Protection Policies enforce data security inside apps using the Intune SDK — no device enrollment required.

🔧
Intune SDK Integration
Microsoft 365 apps (Outlook, Teams, SharePoint, OneDrive, Edge) have the Intune SDK built in. For custom apps, developers integrate the SDK (iOS: Swift/ObjC, Android: Java). SDK intercepts data operations and enforces IT policies.
📋
What APP Policies Control
🚫 Block copy/paste to unmanaged apps
🚫 Block Save As to personal storage (iCloud, Google Drive)
🚫 Block screen capture (Android)
🔒 Require PIN or biometric to open app
⏱️ Offline grace period before access revoked
🧹 Selective wipe on demand or on departure
MAM-WE enrollment model
🔄
SDK Policy Refresh Flow
User signs into managed app → SDK calls Intune with MSAL token → Intune returns App Protection Policy JSON → SDK enforces locally. Policy refreshes every 30 minutes in foreground and on each app launch after 30 min background.
MSAL token flow
📱
Managed Browser — Edge
Microsoft Edge with Intune SDK acts as the managed browser. Links from managed apps (Outlook → SharePoint link) open in Edge, maintaining the managed data boundary. Blocks corporate links opening in Safari or Chrome.
🛡️
Conditional Launch Checks
APP enforces conditional launch: block jailbroken devices, require minimum app version, require minimum OS version, require device PIN. If checks fail: warn, wipe, or block access entirely.
Jailbreak detection
🏢
MAM Without Enrollment
Works without device enrollment. User signs into Microsoft app with work account — policies apply automatically via MSAL token exchange. No Company Portal, no IT device footprint. Ideal for personal devices and contractors.
Lifecycle
App Lifecycle Management
🎯
Required vs Available vs Uninstall
Required: Force installs silently on targets. No user choice.
Available: Appears in Company Portal. User chooses to install.
Uninstall: Removes the app from targeted devices.
📊
Installation Status Reporting
Per-device and per-user install status: Installed, Not Installed, Failed, In Progress, Not Applicable. IME logs provide detailed error codes for Win32 failures.
Query install status via Graph API
🔄
App Updates
Win32: use supersedence for auto-update. Microsoft Store: WinGet auto-updates. iOS VPP: new version pushed automatically. Android Managed Play: IT controls update timing (immediate, postponed up to 90 days, or windowed).
🏗️
Company Portal
Self-service app store for end users. Shows Available apps on demand. Shows device compliance status. Required for BYOD enrollment on most platforms. Available on Windows, iOS, Android, macOS.
BYOD enrollment
🔍
Discovered Apps Inventory
Intune auto-inventories all installed apps on managed devices (Windows, macOS, iOS, Android). Useful for software asset management, licence compliance, and identifying unapproved software.
IT visibility comparison
🗑️
App Retirement & Wipe
Change assignment to Uninstall for graceful removal, or trigger selective wipe to remove app data immediately (for MAM-protected apps). Removes all corporate data from the app without touching the device.
Automate selective wipe via Graph
