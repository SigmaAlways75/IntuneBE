Why Certificates?
Passwordless Device Authentication

Certificates replace passwords for device and user authentication. A managed device proves its identity to Wi-Fi, VPN, and email using a certificate issued by your CA — with no user password prompt.

SCEP
Simple Certificate Enrollment Protocol
Device generates its own key pair — private key never leaves the device
Key Generation
Private key generated on-device, never transmitted. TPM-bound on Windows — hardware-tied.
Infrastructure
Requires NDES (Network Device Enrollment Service) on-premises + Intune Certificate Connector
Security
Highest — private key never transmitted, TPM-protected, hardware-bound
Best For
Wi-Fi auth, VPN auth, high-security device identity
Platforms
Windows, iOS, Android, macOS, Linux
PKCS #12
Public Key Cryptography Standards
CA generates key pair and exports PFX — delivered by Intune
Key Generation
CA generates both public and private key. PFX container delivered to device encrypted with device's public key.
Infrastructure
Intune Certificate Connector only — simpler than SCEP, no NDES required
Security
Good — PFX encrypted in transit, but private key passes through Intune service
Best For
S/MIME email signing/encryption, simpler deployments
Platforms
Windows, iOS, Android, macOS
🏛️
Trusted Root Certificate Profile
Before deploying SCEP or PKCS profiles, push the Trusted Root CA certificate to all devices. Intune delivers root cert via a "Trusted certificate" profile — prerequisite for SCEP/PKCS profiles.
🔌
Intune Certificate Connector
Windows service on an on-prem server bridging Intune cloud to your CA. For SCEP: proxies requests to NDES. For PKCS: communicates with CA directly. Requires outbound HTTPS only — no inbound firewall rules.

☁️
Cloud PKI (Intune Suite)
Microsoft's fully cloud-hosted CA — no on-premises infrastructure. Create root CA and issuing CA entirely in Intune. Issue SCEP and PKCS certificates without any connectors. Available with Intune Suite add-on licence.
SCEP Flow
Complete SCEP Certificate Request Flow

From policy assignment to certificate installed on device — every step of the SCEP process.

🔐 SCEP Flow — Windows Device Example
1
Policy Assignment
Admin creates SCEP profile: subject name, key size (2048-bit), key usage, EKU (Client Authentication), validity period, NDES URL. Assigned to device or user group.
2
Device Receives Profile
At next MDM check-in, device receives the SCEP certificate profile. Windows MDM client reads SCEP parameters and initiates key generation.
3
Key Pair Generated On-Device
Device generates RSA 2048-bit key pair. On Windows with TPM, private key generated inside TPM — it can never be exported. A Certificate Signing Request (CSR) is created with the public key.
4
Intune Issues Challenge Password
Device requests a SCEP challenge password from Intune. Intune verifies device identity (must be enrolled). Calls NDES connector to get a one-time challenge password. Returns it to device.
5
CSR + Challenge → NDES
Device submits CSR and challenge password directly to NDES URL over HTTPS. NDES validates the challenge, forwards certificate request to the internal CA (ADCS).
6
CA Issues Certificate
ADCS processes the request using the assigned certificate template. Signs certificate with CA's private key. Returns signed certificate to NDES → device.
7
Certificate Installed & Reported
Device installs certificate in the appropriate store (Computer or User). Intune notified of successful delivery. Certificate thumbprint and expiry recorded in Intune device record. Ready for Wi-Fi/VPN auth.

PKCS Flow
PKCS #12 Certificate Delivery

How PFX certificates are generated by the CA and securely delivered to devices via the Intune Certificate Connector.

1
PKCS Profile Assigned
Admin creates PKCS profile: CA server, certificate template name, subject name format, target (user or device). Assigns to group.
2
Certificate Connector Polls Intune
The Intune Certificate Connector (on-prem) polls Intune every few minutes for new PKCS certificate requests. No inbound connections — connector initiates outbound HTTPS only.
3
Connector Requests Certificate from CA
Connector calls ADCS via DCOM/RPC to request a certificate using the specified template. CA generates both private key and certificate, exports as PFX encrypted container.
4
PFX Encrypted and Uploaded
Connector encrypts the PFX using the device's Entra ID public key (only the target device can decrypt). Uploads encrypted PFX blob to Intune. Private key never stored in Intune in plaintext.
5
Device Downloads and Installs
At next MDM check-in, device downloads the encrypted PFX from Intune. Decrypts using its own private key. Installs certificate in specified store. Reports back to Intune.
Use Cases
Where Intune-Deployed Certificates Are Used
📶
802.1X Wi-Fi Authentication
WPA2-Enterprise (EAP-TLS). Device presents SCEP certificate to RADIUS server (NPS) for authentication. No password needed. Only Intune-managed devices with a valid cert can join the network.
Wi-Fi profile in Settings Catalog
🌐
VPN Certificate Authentication
VPN gateway (Cisco AnyConnect, Palo Alto GlobalProtect, Microsoft Tunnel) authenticates devices using SCEP certificates. Proves "this is a managed, trusted device." No VPN password prompts.
Microsoft Tunnel & VPN
📧
S/MIME Email Signing & Encryption
PKCS certificates used for S/MIME in Outlook. Signing cert proves email authenticity. Encryption cert enables end-to-end encrypted email. Deployed silently via Intune — no user setup needed.
🔐
MDM Mutual TLS
After initial enrollment, Windows devices use a client certificate for all MDM check-ins with Intune. Replaces initial token-based auth. TPM-bound on Windows — hardware-tied to that specific device.
☁️
Entra ID Certificate-Based Auth (CBA)
Use Intune-issued client certs as a phishing-resistant MFA factor for Conditional Access — hardware-bound authentication without FIDO2 keys.
CA policy for CBA
🖨️
Network Resource Access
Printers, NAS devices, and internal web apps requiring certificate authentication. Devices with Intune-issued certs access these resources seamlessly — without users entering domain credentials.
