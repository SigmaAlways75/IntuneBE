Control Plane
Graph API — The Intune Control Plane

The Intune portal is a UI wrapper over Microsoft Graph API. Every button click, policy save, and device action in the portal is a Graph API call. Automating Intune means automating Graph.

🏗️
Everything is Graph
Create policies, assign apps, query compliance, trigger remote actions, read reports — all via REST calls to graph.microsoft.com. The portal, PowerShell module, and all third-party tools use the same API surface.
🔐
App Registration Required
Register an App in Entra ID. Assign DeviceManagement.* permissions. Use client credentials (app secret or certificate) for unattended automation.
Entra ID app registration
📦
SDK Support
Official Microsoft Graph SDKs for PowerShell, Python, .NET, JavaScript, Java, Go. The PowerShell SDK (Microsoft.Graph module) is the most common for Intune automation.
🔔
Webhooks & Change Notifications
Subscribe to change notifications for when device compliance changes, a device enrolls, or a policy is modified. Triggers Logic Apps or Azure Functions for event-driven automation.
📊
Reporting & Analytics
Pull device inventory, compliance reports, app install status via Graph. Feed into Power BI for dashboards. Schedule exports to Azure Blob Storage via Logic Apps.
Visibility & analytics
🌍
Multi-Tenant Management
MSPs can manage multiple customer tenants via Graph API using Lighthouse or custom multi-tenant app registrations. Deploy identical baseline policies across hundreds of tenants via a single script.
Authentication
Getting a Graph API Token
🔐 Client Credentials Flow (Unattended)
# PowerShell — Get token using App Registration
$body = @{
    grant_type    = "client_credentials"
    client_id     = $clientId
    client_secret = $clientSecret  # Use certificate in production
    scope         = "https://graph.microsoft.com/.default"
}
$token = (Invoke-RestMethod -Uri "https://login.microsoftonline.com/$tenantId/oauth2/v2.0/token" -Method POST -Body $body).access_token
⚡ Microsoft Graph PowerShell SDK (Recommended)
# Install
Install-Module Microsoft.Graph -Scope CurrentUser

# Connect interactive (delegated)
Connect-MgGraph -Scopes "DeviceManagementConfiguration.ReadWrite.All"

# Connect app-only (unattended) — use cert, not secret
Connect-MgGraph -TenantId $tenantId -ClientId $clientId -CertificateThumbprint $certThumb

# Use Graph cmdlets
Get-MgDeviceManagementManagedDevice | Select-Object DeviceName, ComplianceState
🔑
Required Permissions
Read policies: DeviceManagementConfiguration.Read.All
Write policies: DeviceManagementConfiguration.ReadWrite.All
Read devices: DeviceManagementManagedDevices.Read.All
Remote actions: DeviceManagementManagedDevices.PrivilegedOperations.All
Apps: DeviceManagementApps.ReadWrite.All
📜
Certificate vs Secret
For production automation, always use certificate authentication. Store certificates in Azure Key Vault, rotate programmatically. Secrets expire and can be exposed in logs.

PowerShell Scripts
Common Automation Scripts
💾 Export All Compliance Policies (Backup)
Connect-MgGraph -Scopes "DeviceManagementConfiguration.Read.All"
$policies = Get-MgDeviceManagementDeviceCompliancePolicy -All
foreach ($p in $policies) {
    $detail = Get-MgDeviceManagementDeviceCompliancePolicy -DeviceCompliancePolicyId $p.Id
    $name = $p.DisplayName -replace '[\/:*?"<>|]', '_'
    $detail | ConvertTo-Json -Depth 20 | Out-File ".ackup\$name.json"
    Write-Host "Exported: $($p.DisplayName)"
}
📊 Export Non-Compliant Devices to CSV
Connect-MgGraph -Scopes "DeviceManagementManagedDevices.Read.All"
$nonCompliant = Get-MgDeviceManagementManagedDevice -All `
    -Filter "complianceState eq 'noncompliant'" `
    -Select "deviceName,userPrincipalName,complianceState,osVersion,lastSyncDateTime"
$nonCompliant | Export-Csv "NonCompliant-$(Get-Date -f yyyyMMdd).csv" -NoTypeInformation
Write-Host "Found $($nonCompliant.Count) non-compliant devices"
🔄 Trigger Sync on All Non-Compliant Devices
Connect-MgGraph -Scopes "DeviceManagementManagedDevices.ReadWrite.All", `
    "DeviceManagementManagedDevices.PrivilegedOperations.All"
$devices = Get-MgDeviceManagementManagedDevice -All -Filter "complianceState eq 'noncompliant'"
foreach ($d in $devices) {
    Invoke-MgSyncDeviceManagementManagedDevice -ManagedDeviceId $d.Id
    Write-Host "Sync triggered: $($d.DeviceName)"
    Start-Sleep -Milliseconds 300 # rate limiting
}
Infrastructure as Code
GitOps & IaC for Intune

Manage Intune configuration declaratively — version-controlled, auditable, and deployable across multiple tenants.

🏗️
Terraform Intune Provider
The microsoft365 Terraform provider manages Intune resources declaratively in HCL. terraform apply deploys to any tenant. terraform plan shows drift.
Policy types reference
📂
IntuneCD — Backup & Restore
Open-source Python tool exporting all Intune config to JSON/YAML and importing it back. For tenant migration, disaster recovery, multi-tenant baseline deployment. Integrates with Azure DevOps and GitHub Actions.
🔄
GitHub Actions Pipeline
Commit policy changes to Git → GitHub Actions calls Graph API to apply. Pull requests create a review process. Tags trigger versioned deployments. Full audit trail of every policy change in Git history.
⚡
Azure Logic Apps — Event Automation
New device enrolled → auto-assign to group based on serial number prefix. User disabled → trigger selective wipe on all their devices. Policy modified → send Teams alert to security channel.
🔧 Terraform Example — Compliance Policy
resource "microsoft365_device_compliance_policy" "windows_standard" {
  display_name  = "Windows - Standard Compliance"
  platform_type = "Windows10"

  windows10_compliance {
    os_minimum_version     = "10.0.19041"
    bit_locker_enabled     = true
    secure_boot_enabled    = true
    defender_enabled       = true
    password_minimum_length = 12
  }

  assignments {
    target { group_id = var.all_devices_group_id }
  }
}
