Three VPN Approaches in Intune

Intune manages three VPN approaches — Microsoft Tunnel (Microsoft's own mobile gateway), Per-App VPN (route only specific apps), and Always-On VPN (Windows full-device tunnel).

🚇
Microsoft Tunnel
Purpose-built VPN gateway for iOS and Android. Runs as a Docker container on Linux in your DMZ. No third-party VPN licence needed. Integrates with Conditional Access for compliance gating.
Zero Trust network pillar
📱
Per-App VPN
Only specified apps route traffic through the VPN. Personal apps go direct to internet. Work apps tunnel to on-premises. Works with Tunnel or third-party VPNs.
App association
💻
Always-On VPN (Windows)
Windows 10/11 built-in VPN. Connects automatically before user login and stays connected. Uses IKEv2 or SSTP. Configured via Intune DeviceTunnel + UserTunnel CSP profiles.
VPN profile in Settings Catalog
🔌
Third-Party VPN Integration
Intune supports VPN profiles for Cisco AnyConnect, Palo Alto GlobalProtect, Zscaler, Check Point, F5, and others. Intune delivers config — the VPN client app deployed via app management.
Deploying VPN client apps
🔑
Certificate Authentication for VPN
All Intune VPN integrations support certificate-based authentication. SCEP certificate deployed by Intune authenticates the device — no passwords, phishing-resistant.
SCEP certificate delivery
🚦
Conditional Access + VPN
Only compliant, Intune-managed devices can establish a VPN connection. Non-compliant devices denied at the gateway using certificate validation integrated with Intune compliance state.
CA compliance check
Tunnel Architecture
Microsoft Tunnel — How It Works

Microsoft Tunnel runs as a Docker container on a Linux server in your DMZ, providing managed iOS and Android devices with secure access to on-premises resources.

🔄 Tunnel Connection Flow
1
Device Requests Connection
Microsoft Defender app on iOS/Android initiates connection to the Tunnel gateway. Intune VPN profile specifies the Tunnel server FQDN and port 443. Defender acts as the VPN client — no separate app needed.
2
Mutual TLS Authentication
Device presents its Intune-managed client certificate (from SCEP/PKCS). Tunnel server validates certificate against the trusted root CA. Device validates Tunnel server's TLS certificate. mTLS established.
Certificate deployment
3
Compliance Check via Intune
Tunnel server calls Intune service to verify the device is compliant. Non-compliant devices are rejected — even with a valid certificate. This is the Conditional Access integration point for Tunnel.
4
TLS Tunnel Established
Specified app traffic (per-app VPN) or all traffic routes through the tunnel. Tunnel server proxies requests to on-premises resources on the internal network.
5
Tunnel Forwards to On-Prem
Tunnel server accesses the internal network via its second NIC or routing. Forwards device requests to appropriate internal servers. Returns responses through the tunnel to the device.
Per-App VPN
Route Only Work Apps Through VPN

Per-App VPN ensures only corporate apps use the VPN tunnel. Personal apps go directly to the internet. Configured by associating specific apps with a VPN profile in Intune.

📱 How Per-App VPN Works on iOS
1. Admin creates VPN profile in Intune (Tunnel or third-party) with per-app VPN mode enabled
2. Admin associates specific managed apps with the VPN profile
3. Policy deployed to device — iOS registers app bundle IDs with the VPN framework
4. User opens Teams or SharePoint → iOS automatically brings up the VPN tunnel
5. User opens Safari or Instagram → traffic goes directly to internet, no VPN
6. VPN disconnects automatically when all associated apps are closed

Result: Corporate data never leaves the tunnel. Personal data never touches the corporate network.
BYOD data isolation
📲
App Association in Intune
In the VPN profile assignment, specify which apps trigger VPN using bundle IDs. Works with Intune-deployed apps. On Android, works with Work Profile apps only — personal side excluded by design.
✂️
Split Tunneling
Even in full-device VPN mode, route only specific IP ranges or domains through the tunnel. Internet traffic goes direct. Reduces bandwidth on VPN gateway and improves user performance.
🔒
On-Demand VPN Rules (iOS)
Automatically connect when accessing specific domains (*.contoso.com), disconnect when on trusted Wi-Fi (corporate Wi-Fi). Configured via Intune VPN profile XML payload.

Windows Always-On VPN

Windows Always-On VPN maintains a persistent tunnel for Windows devices, providing seamless on-premises access without user interaction.
